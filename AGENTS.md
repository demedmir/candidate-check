# AGENTS.md — инструкция для AI-агентов

Этот документ адресован агентам (Claude / Codex / Cursor / Aider / OpenCode), которые
будут дорабатывать candidate-check. Задача написана так, чтобы небольшая модель могла
работать с одним модулем не загружая в контекст всё репо.

## Принципы

1. **Один модуль — одна задача.** Не редактируй несколько слоёв сразу.
   Backend и frontend — отдельные сессии. Микросервисы (`services/*`) — отдельные.
2. **Шаблоны.** Перед написанием нового коннектора / страницы / миграции —
   прочитай файл-шаблон в `templates/` и **скопируй**. Не изобретай заново.
3. **Не трогай миграции, которые уже применены.** Если нужно изменение схемы —
   создай новую миграцию `alembic/versions/000N_<имя>.py` (новый номер).
4. **Adjudication — конфиг, а не код.** Поменять веса = `config/adjudication.yaml`,
   не Python.
5. **Перед коммитом**:
   - Backend: `docker compose run --rm app python -m pytest tests/ -q`
   - Frontend: `cd frontend && npm run build`
6. **Сообщение коммита** — conventional: `feat(connectors): ...`, `fix(api): ...`,
   `refactor(ui): ...`. На русском или английском — единообразно в рамках одного PR.

## Что строго НЕЛЬЗЯ

- **Парсить «пробив-боты», утечки, телеграм-боты с ПДн** — статья 137/272 УК.
- **Хардкодить секреты** в коде или `.env.example`. Только переменные.
- **Скачивать** что-либо в обход `app/scripts/refresh_*.py` (так у нас единая точка
  обновления локальных датасетов, и `cron` подхватит автоматически).
- **Удалять `tests/` или конкретный тест** без разговора с человеком.
- **Чистить `*pycache*`/`dist/`** в коммитах — они в `.gitignore`.

## Запрос помощи

Тип сообщения: «Я собираюсь сделать X, есть нюанс Y, продолжать?».
