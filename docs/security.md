# Security & Compliance

## 152-ФЗ (Персональные данные)

### Согласие на обработку ПДн

В MVP реализован **бумажный flow**: HR при создании кандидата отмечает чекбокс
«согласие подписано на бумаге» и опционально аплоадит скан. **Без отметки запуск
проверки заблокирован** на уровне API (`HTTPException 400`).

⚠️ Для production-режима с массовыми проверками рекомендуется юр-значимый
электронный flow:
- IDX (https://idx.ru) — оператор согласий через ЕСИА/Sber ID/SMS
- Своя реализация ЕСИА-flow (требует аккредитации)
- СМС-подпись (простая ЭП по 63-ФЗ)

### Категории данных

В системе обрабатываются:

| Категория | Что | Где |
|---|---|---|
| **Общие ПДн** | ФИО, ДР, ИНН, СНИЛС, телефон, email | `candidates` |
| **Идентификационные** | паспорт, скан паспорта | `candidates`, `candidate_documents` |
| **Специальные категории** | справка ПНД (психиатрия), нарко-диспансер | `candidate_documents` (только при добровольном предоставлении кандидатом) |
| **Биометрия** | потенциально — фото в распознавании паспорта | временно, не сохраняется |
| **Сведения о судимости** | справка о судимости (от Госуслуг кандидата) | `candidate_documents` |

### Категории субъектов

- **HR-пользователь** — сотрудник работодателя, имеет JWT
- **Кандидат** — субъект ПДн, физически не входит в систему (бумажное согласие)

## Аутентификация и авторизация

### JWT для SPA

- Алгоритм: HS256
- Secret: `APP_SECRET` (32-байтный random)
- TTL: 24 часа
- Хранение: localStorage (Zustand `persist`)
- Передача: `Authorization: Bearer <token>`

⚠️ **localStorage уязвим к XSS**. Для production-усиления:
- Cookie httpOnly + Same-Site Strict
- Refresh-token rotation
- Короткий access-TTL (15 мин)

### Session для legacy HTMX

- itsdangerous TimestampSigner
- HttpOnly cookie `ccsession`
- Same-Site Lax
- TTL = `HR_SESSION_TTL_SECONDS` (24h по умолчанию)

### Пароли

- Argon2id (passlib + argon2-cffi)
- Соль автоматическая
- Время `time_cost=3, memory_cost=64MB, parallelism=4`

## Передача данных

| Сегмент | Защита |
|---|---|
| Browser ↔ nginx | TLS 1.2/1.3 (Let's Encrypt) |
| nginx ↔ FastAPI | localhost (loopback) |
| FastAPI ↔ Postgres | docker network (изолирован) |
| FastAPI ↔ Redis | docker network |
| FastAPI ↔ Spark OCR | **Tailscale WireGuard** (encrypted) |
| Spark ↔ публичные источники | TLS |

## Управление секретами

`.env` — НЕ в git (`.gitignore`). Содержит:
- `APP_SECRET` — JWT-ключ + cookie-signer
- `POSTGRES_PASSWORD`
- `CAPTCHA_API_TOKEN` (опционально, на solver)
- `OCR_API_TOKEN` (опционально, на passport-ocr)

При компрометации: ротация = смена `APP_SECRET` инвалидирует все JWT и сессии.

⚠️ **Секреты в логи не пишутся**. structlog не логирует password/token поля.

## CORS

В dev: `localhost:5173` (vite). В production: same-origin (фронт раздаётся nginx
с того же домена), CORS не критичен.

## Загрузка файлов

- Лимит размера: 20 MB (nginx `client_max_body_size`)
- MIME-проверка: на frontend (accept). На backend — не проверяем, любой бинарник.
- Хранение: `/data/uploads/` (docker volume), permissions 644 от root.

⚠️ **Не реализовано** (TODO для production):
- Антивирус-скан загруженных файлов (ClamAV)
- Quarantine + manual review для крупных
- Шифрование at-rest (LUKS / SQL TDE)

## Логирование

structlog JSON, попадает в `docker logs`. Автоматически в логи:
- HTTP-метод/path/code (uvicorn)
- worker job-ID/connector-ключ/duration

Не логируются:
- Пароли
- JWT-токены
- Содержимое payload коннекторов (только summary)
- Содержимое загруженных файлов

## Известные ограничения текущего MVP

1. Single-tenant — все HR видят всех кандидатов (нет ролевой модели сейчас)
2. Нет rate-limit на API (потенциально DoS-уязвим)
3. Нет audit log пользовательских действий (только в БД-полях)
4. Нет автоудаления ПДн по сроку (надо реализовать retention policy)
5. JWT в localStorage (XSS-уязвимость, см. выше)
